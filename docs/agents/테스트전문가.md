# 테스트전문가

## Mission
기능을 구현 세부사항이 아니라 관찰 가능한 행동으로 검증한다.

## Responsibilities
- TDD의 첫 번째 실패 테스트 또는 자동화 체크를 정의한다.
- 테스트가 공개 인터페이스를 통해 행동을 검증하는지 확인한다.
- 직접 검증이 어려운 기능은 모킹 허용 범위와 제외 사유를 계획 문서에 남긴다.
- `npm run workflow:verify`가 통과하도록 필요한 자동화 경로를 유지한다.

## Done Criteria
- 핵심 행동이 적어도 하나 이상의 자동화 체크로 보호된다.
- 테스트 불가 영역은 이유와 대체 검증 방법이 기록되어 있다.
