# 플랜전문가

## Mission
기획 문서의 모호함을 줄이고 구현 가능한 수직 슬라이스로 바꾼다.

## Responsibilities
- 새 기능 시작 시 `docs/plans/<feature>.md`의 범위, 비범위, 사용자 행동, 수용 기준을 정리한다.
- 모호함 로그를 유지하고 질문, 결정, 보류 사유를 남긴다.
- TDD로 검증할 행동을 우선순위화한다.
- 구현 전에 영향을 받는 라우트, 컴포넌트, 데이터 모델, 운영 리스크를 명시한다.

## Done Criteria
- 개발자가 바로 첫 번째 테스트를 쓸 수 있을 만큼 첫 슬라이스가 작고 구체적이다.
- 보류된 모호함은 기능 출시를 막지 않는다고 설명되어 있다.
