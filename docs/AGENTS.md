# Documentation Guidelines

## Scope
Applies to `docs/`, including feature plans, process guides, architecture notes, and specialist agent documents.

## Rules
- Every feature starts with a plan in `docs/plans/` generated by `npm run workflow:plan -- <feature-slug>` or `npm run workflow:new -- <feature-slug>`.
- `npm run workflow:new -- <feature-slug>` must create or switch to `feature/{YYYYMMDD}_{feature-slug}`.
- Plans must keep an ambiguity log. Resolve open questions through a loop before implementation.
- Use concrete acceptance checks instead of vague intent.
- Keep specialist agent documents in `docs/agents/` and update them when workflow responsibilities change.
- Keep machine-readable multi-agent workflow contracts in `.ai/`.

## Plan Minimums
- Scope and non-goals.
- User-facing behavior.
- TDD or automation strategy.
- Test-first implementation order, with Test Agent before Frontend Agent.
- Playwright MCP QA path for UI changes.
- Commit and rollout notes.
