# DAITJI 임팩트 큰 기능 로드맵

## 목적

현재 DAITJI는 개인 재고/위치 관리 앱의 핵심 흐름이 구축된 상태다.
다음 단계에서는 기능 수를 늘리는 것보다, 사용자가 자주 돌아오고 계속 기록하게 만드는 방향으로 우선순위를 잡는 것이 중요하다.

이 문서는 `1주`, `2주`, `1달` 단위로 구현 임팩트가 큰 기능 로드맵을 제안한다.

---

## 1주 로드맵

### 목표

기존 기능을 크게 흔들지 않으면서 사용자가 바로 체감할 수 있는 개선을 만든다.

### 추천 범위

#### 1. 스캔 결과 기반 빠른 등록 보조

- 바코드 스캔 후 같은 코드의 기존 등록 정보 제안
- 이름, 타입, 보관 위치 자동 프리필
- 반복 등록 UX 개선

#### 2. 즐겨찾기 위치 / 물품

- 자주 보는 위치를 빠르게 접근 가능하게 구성
- 대시보드와 탐색 화면에 즉시 반영

#### 3. 대시보드에 알림 요약 섹션 추가

- 3일 이내 만료
- 7일 이내 만료
- 재고 부족 물품

### 기대 효과

- 사용자가 앱의 속도와 편의성을 바로 체감할 수 있다.
- 기존 `scan`, `dashboard`, `explorer` 흐름을 자연스럽게 강화한다.
- 기능 대비 구현 범위가 비교적 작다.

### 성공 기준

- 스캔 후 물품 등록 시간이 줄어든다.
- 자주 보는 위치 접근 수가 늘어난다.
- 대시보드에서 행동 가능한 항목이 더 잘 보인다.

---

## 2주 로드맵

### 목표

운영 효율과 재방문 빈도를 높여 "계속 쓰는 앱"으로 만드는 기반을 강화한다.

### 추천 범위

#### 1. 알림 센터 페이지

- 오늘 확인할 항목 목록화
- 만료 임박 / 재고 부족 구분
- 향후 읽음 처리나 snooze 기능 확장 가능하도록 설계

#### 2. 최근 본 물품 / 위치

- 대시보드와 탐색 화면에서 재진입성 강화
- 모바일 사용 시 탐색 손실을 줄여줌

#### 3. 대량 선택 후 일괄 액션

- 여러 물품을 한 번에 이동, 삭제, 상태 변경
- 실제 정리 작업 시간을 줄여줌

### 기대 효과

- 알림이 단순 설정이 아니라 실제 행동 흐름으로 연결된다.
- 품목 수가 늘어나도 관리 피로도가 완만해진다.
- 앱이 "조회형"에서 "운영형"으로 한 단계 올라간다.

### 성공 기준

- 알림 화면에서 상세 진입률이 발생한다.
- 다건 편집 작업 시간이 눈에 띄게 줄어든다.
- 최근 본 항목을 통해 같은 세션 내 재탐색 비용이 줄어든다.

## 1달 로드맵

### 목표

앱의 확장성과 차별점을 만들 수 있는 기능 축을 확보한다.

### 추천 범위

#### 1. 재고 액션 히스토리

- 생성, 수정, 이동, 수량 변경, 삭제 이력 기록
- 누가 언제 무엇을 바꿨는지 확인 가능
- 향후 공유 기능과도 잘 연결됨

#### 2. 아이템 상세 page-first 정리 + 공유 액션

- `/item/[id]`를 canonical detail로 강화
- 링크 복사, 공유 진입 기반 확보
- modal은 quick view 역할로 축소

#### 3. 쇼핑리스트 연동

- 재고 부족 알림에서 바로 쇼핑리스트 추가
- 구매 완료 시 재고 반영
- 일상 사용 루틴과 연결

#### 4. 구독/플랜 기반 준비 작업

- 문서에 있는 `Free / Pro / Family` 설계를 실제 엔드포인트와 설정 화면으로 연결
- 우선 `현재 플랜`, `사용량`, `제한` 조회 화면부터 제공

### 기대 효과

- 데이터 신뢰성과 운영 히스토리가 생긴다.
- 외부 공유와 협업 확장 가능성이 열린다.
- 앱이 단순 보관 관리에서 생활 관리 도구로 확장된다.

### 성공 기준

- 상세 페이지가 canonical entry로 안정화된다.
- 사용자가 "기록한 데이터가 쌓인다"는 느낌을 받는다.
- 장기적으로 유료화/가족 공유로 연결할 수 있는 구조가 준비된다.

---

## 추천 실행 순서

### 가장 현실적인 순서

1. 1주: 스캔 등록 보조 + 즐겨찾기 + 대시보드 알림 요약
2. 2주: 알림 센터 + 최근 본 항목 + 대량 선택 액션
3. 1달: 이력 관리 + 상세 공유 구조 + 쇼핑리스트 + 플랜 준비

---

## 제품 관점 제안

현재 단계에서 가장 중요한 축은 아래 3가지다.

1. `입력 속도 개선`
2. `확인할 일 가시화`
3. `쌓이는 데이터의 가치 강화`

각 축에 대응되는 대표 기능은 다음과 같다.

- 입력 속도 개선: 스캔 기반 등록 보조
- 확인할 일 가시화: 알림 센터
- 쌓이는 데이터의 가치 강화: 재고 액션 히스토리

---

## 메모

- 단기 목표는 "앱이 더 편해졌다"를 만드는 것이다.
- 중기 목표는 "앱에 다시 들어올 이유"를 만드는 것이다.
- 장기 목표는 "혼자 쓰는 관리 앱"에서 "함께 쓰거나 오래 쌓아두는 시스템"으로 확장하는 것이다.
