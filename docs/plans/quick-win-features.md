# DAITJI 빠르게 만들 수 있는 기능 5개

## 목적

현재 코드베이스는 `items`, `explorer`, `dashboard`, `scan`, `settings/notifications`의 기본 흐름이 이미 갖춰져 있다.
따라서 다음 기능은 완전히 새로운 도메인을 여는 것보다, 기존 흐름에 얹어서 짧은 구현으로 체감 가치를 크게 만드는 방향이 효율적이다.

이 문서는 현재 구조를 기준으로 빠르게 구현 가능하면서 사용자 만족도를 높이기 좋은 기능 5개를 정리한다.

---

## 1. 즐겨찾기 위치 / 물품

### 왜 먼저 하면 좋은가

- `explorer`, `dashboard` 탐색 빈도를 줄여준다.
- 자주 찾는 위치를 한 번에 열 수 있어 반복 사용성이 높아진다.
- UI 추가 범위가 비교적 작고 이해하기 쉬운 기능이다.

### 사용자 가치

- 냉장고, 욕실, 약 보관함 같은 자주 보는 위치를 바로 열 수 있다.
- 자주 확인하는 물품을 별도로 모아볼 수 있다.

### 구현 범위

- 위치/물품에 `favorite` 상태 저장
- 카드 또는 상세 화면에 즐겨찾기 토글 버튼 추가
- 대시보드 상단에 `즐겨찾기` 섹션 추가
- Explorer 또는 Items 페이지에 `즐겨찾기만 보기` 필터 추가

### 추천 진입점

- `components/features/LocationGridCard.tsx`
- `components/features/ItemCard.tsx`
- `app/dashboard/DashboardClient.tsx`

## 2. 최근 본 물품 / 위치

### 왜 먼저 하면 좋은가

- 별도 복잡한 비즈니스 로직 없이도 사용성이 좋아진다.
- 상세 패널과 탐색 흐름이 이미 있어 연결이 쉽다.
- 저장 위치를 클라이언트 로컬 상태로 시작할 수 있어 구현 부담이 작다.

### 사용자 가치

- 방금 봤던 물품으로 빠르게 돌아갈 수 있다.
- 탐색 중 길을 잃지 않고 이어서 확인할 수 있다.

### 구현 범위

- 최근 본 위치/물품 목록을 `localStorage` 또는 사용자별 테이블에 저장
- 대시보드에 `최근 본 항목` 섹션 추가
- 최대 5~10개까지만 유지

### 추천 진입점

- `app/explorer/ExplorerClient.tsx`
- `app/item/[id]/ItemDetailPageClient.tsx`
- `app/dashboard/DashboardClient.tsx`

## 3. 스캔 결과 기반 빠른 등록 보조

### 왜 먼저 하면 좋은가

- 이미 바코드 스캔과 물품 추가 화면 연결이 구현되어 있다.
- 사용자가 가장 즉각적으로 "편해졌다"고 느낄 가능성이 크다.
- 신규 화면보다 기존 흐름 개선 중심이라 리스크가 낮다.

### 사용자 가치

- 같은 바코드의 등록 이력이 있으면 이름, 타입, 기본 위치를 자동 제안받을 수 있다.
- 반복 등록 시간이 크게 줄어든다.

### 구현 범위

- `barcode` 기준 최근 등록 정보 조회 API 또는 클라이언트 조회
- `/items/add?barcode=...` 진입 시 기존 등록 이력 제안
- `이전 정보 불러오기` 버튼 또는 자동 프리필

### 추천 진입점

- `app/scan/ScanClient.tsx`
- `app/items/add/ItemAddClient.tsx`
- `app/api/items/route.ts`

## 4. 알림 센터 요약 화면

### 왜 먼저 하면 좋은가

- 알림 설정 및 알림 큐 시스템 문서가 이미 존재한다.
- 현재는 설정 중심이라 사용자 입장에서 실행 가능한 액션 화면이 부족하다.
- 대시보드 또는 별도 페이지 하나로 확장 가능하다.

### 사용자 가치

- 오늘 확인해야 할 만료 임박 물품과 재고 부족 물품을 한 곳에서 본다.
- 단순 설정이 아니라 실제 행동으로 이어진다.

### 구현 범위

- `만료 임박`, `재고 부족` 묶음 섹션 추가
- 읽음/확인 상태는 후순위로 두고 우선 목록 조회부터 구현
- 대시보드에서 `전체 보기` 링크 제공

### 추천 진입점

- `app/dashboard/DashboardClient.tsx`
- `app/api/items/expiring/route.ts`
- 신규 `app/notifications/page.tsx` 또는 dashboard section

## 5. 대량 선택 후 일괄 이동 / 삭제 / 상태 변경

### 왜 먼저 하면 좋은가

- 물품 개수가 늘어날수록 운영 효율 차이가 크게 난다.
- 기존 item list 구조 위에 selection state를 추가하는 형태로 접근 가능하다.
- 고급 기능처럼 보이지만 실제 사용 체감이 강하다.

### 사용자 가치

- 여러 물품을 한 번에 다른 위치로 옮길 수 있다.
- 정리할 때 반복 클릭이 줄어든다.

### 구현 범위

- 리스트에서 multi-select 상태 추가
- 하단 액션 바 또는 bottom sheet 제공
- 우선순위 액션:
  - 위치 이동
  - 삭제
  - 상태 변경

### 추천 진입점

- `components/features/ItemList.tsx`
- `components/features/ItemListRowCard.tsx`
- `app/items/ItemsClient.tsx`
- `app/explorer/ExplorerClient.tsx`

## 우선순위 추천

가장 빠르게 성과를 내기 좋은 순서는 아래와 같다.

1. 스캔 결과 기반 빠른 등록 보조
2. 알림 센터 요약 화면
3. 즐겨찾기 위치 / 물품
4. 최근 본 물품 / 위치
5. 대량 선택 액션

---

## 메모

- 빠른 성공 경험이 중요하면 `스캔 등록 보조`와 `즐겨찾기`가 적합하다.
- 재방문율과 운영성을 높이려면 `알림 센터`와 `대량 선택`의 가치가 크다.
- 서버 스키마 변경을 최소화하고 싶다면 `최근 본`, `즐겨찾기`는 초기 버전을 클라이언트 저장으로 시작할 수 있다.
